Contribute to Practical Bug Bounty

Thank you for your interest in contributing to the Practical Bug Bounty platform! We value the efforts of the community in making this platform a comprehensive resource for bug bounty enthusiasts.

Contributing to the Web Platform

If you have feature suggestions, bug fixes, or any other contributions to the main platform, follow these steps:

  1. Fork the main repository.
  2. Clone your fork and create a new branch for your feature or fix.
  3. Once you've made your changes, submit a pull request to the main repository.
  4. Your changes will be reviewed and merged upon approval.

Contributing to Content

For content contributions, we maintain a separate repository. To contribute to the content:

  1. Fork and clone the content repository.
  2. Create a new module, submodule, or topic by following the folder structure and adding the necessary JSON and markdown files.
  3. Use the provided script to generate a Django-compatible JSON dump.
  4. Submit a pull request with your changes to the content repository.
  5. Your content will be reviewed and, if approved, merged and reflected on the main platform.

Stay Connected

Join our community on Discord, Twitter, or any other platform to stay updated, get support, and collaborate with other contributors.
